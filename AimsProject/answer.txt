Section 9:
The classes that aggregate other classes are Cart, Store, CompactDisc, and Book. Cart contains a list of Media objects, Store contains a list of Media objects, CompactDisc contains a list of Track objects, and Book contains a list of authors. These lists are initialized in the class fields, so the objects can use them immediately after construction.

Section 10:
If the passing object is not an instance of Media, the equals() method should return false. We should check obj == null or !(obj instanceof Media) before casting. Otherwise, casting directly may cause a ClassCastException. For Media, two objects are equal if they have the same title. For Track, two objects are equal if they have the same title and length.

Section 11:
When we store Book, DigitalVideoDisc, and CompactDisc objects in an ArrayList<Media> and call toString(), Java will call the toString() method of the actual object type at runtime. This is polymorphism. Although the reference type is Media, the result depends on whether the object is actually a Book, DVD, or CD.

Section 12:
If we use Comparable, the Media class should implement Comparable<Media> because all items in the cart are Media objects. The compareTo() method can define one default ordering, for example by title first and then by cost. However, Comparable is not suitable when we need multiple sorting rules, such as sorting by title then cost or by cost then title. In that case, Comparator is better. If DVDs need a special ordering rule, we can create a separate Comparator for DigitalVideoDisc or handle DVD-specific comparison inside a custom comparator.
